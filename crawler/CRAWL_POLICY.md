# Crawler 抓取策略

## 目标内容
优先保留以下静态文章或长期有效页面：
- 新手攻略
- build / 职业玩法
- MF / 开荒 / 装备 / 地图 / 掉落 / 符文之语
- FAQ / 常见问题
- 机制说明 / 百科型资料页

## 排除内容
默认排除：
- 交友
- 卖号 / 出号 / 收号
- 交易 / 估价 / 拍卖
- 明显灌水帖
- 与暗黑2百科建设无关的社区杂项页

## 来源优先级
1. 官方站 / 官方中文新闻
2. 中文资料站 / 攻略站
3. 高质量社区精华帖
4. 门户聚合站
5. 代理或人工补数来源

## 增量原则
- 种子页 / 索引页定时抓
- 文章详情页首次抓到就保存快照
- 之后默认冻结，不再重复抓取
- 如人工确认要更新，显式使用 `--refresh-existing`

## 内容关系维护
每个网页路径快照都应维护：
- URL
- 页面标题
- 快照文件路径
- 内容 hash
- 首次发现时间
- 最近刷新时间

对于已冻结的静态页：
- 默认不重复抓正文
- 页面级记录应保持稳定，避免每次调度都改写同一网页文件

对于已保存的正文与详情：
- 目录结构尽量贴近原始站点 host/path
- 详情 JSON 作为正文 HTML 的伴生文件输出，便于静态镜像发布和元数据索引

这样后续无论做自定义正文转换还是向量化，都能从路径目录回溯到原始快照文件。

## 快照清理

```bash
python3 crawler/prune_snapshots.py --keep-runs 10 --prune-snapshots --dry-run
```

用于清理旧 run 目录，以及删除 `page_catalog.json` 不再引用的孤儿快照文件。
