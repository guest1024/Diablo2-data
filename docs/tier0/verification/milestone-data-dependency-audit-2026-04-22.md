# Milestone Data Dependency Audit — 2026-04-22

## Executive summary

Current runtime **does not yet satisfy** the PostgreSQL-oriented target architecture described in the proposal:

- runtime vector layer = `ChromaDB` (`.data/chroma`)
- runtime graph layer = local JSONL (`docs/tier0/merged/*.jsonl`)
- **not** PostgreSQL / `pgvector` / `pg_trgm` / recursive CTE yet

Data-wise, the repository already had decent text coverage, but the structured layer was missing several milestone-critical assets. This audit round added new raw structured sources plus new support assets under `docs/tier0/structured/`.

## Remaining gaps

1. **No PostgreSQL runtime yet**
   - still missing the actual `documents / chunks / aliases / nodes / edges` PG schema
   - still missing `pg_trgm`, `pgvector`, JSONB filters, recursive SQL execution path

2. **Structured facts are not yet imported into the runtime graph**
   - the new `docs/tier0/structured/*.jsonl` layer is ready for ingestion
   - but `docs/tier0/merged/canonical-claims.jsonl` and `support-edges.jsonl` have not yet been rebuilt around these new facts

3. **Build-to-gear-to-farm-route reasoning is still incomplete**
   - `Hammerdin -> core gear -> target farm route` is not fully encoded as explicit graph edges yet
   - current `build-archetypes.jsonl` still has only starter-level build structure

## Recommended next step

If the next objective is to make the proposal fully executable, priority order should be:

1. import `docs/tier0/structured/*.jsonl` into a real PG schema
2. materialize gameplay edges such as:
   - `USES_RUNE`
   - `REQUIRES_BASE`
   - `REQUIRES_SKILL`
   - `DROPS_IN`
   - `USED_BY_BUILD`
3. move community blackwords from term-map-only handling into a first-class alias table suitable for `pg_trgm`
4. rebuild query recipes around recursive graph traversal and numeric table lookup
