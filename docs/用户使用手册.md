# Diablo II 知识库 / 问答系统用户使用手册

这份手册面向使用者，而不是开发者。  
你只需要关心：

1. 当前系统有什么
2. 怎么启动
3. 可以问什么
4. 回答结果怎么理解

---

## 1. 当前系统是什么

当前仓库已经有一个第一版问答底座：

- FastAPI 服务
- Chroma 向量检索
- 本地 graph grounding
- 中英术语映射
- community slang / acronym 稳定召回

它适合：

- Diablo II 百科问答
- 装备 / 符文之语 / 任务 / 地图 / Boss 查询
- 中文黑话 / 英文缩写问答

---

## 2. 快速启动

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

uvicorn app.main:app --reload
```

再导入运行时数据：

```bash
curl -X POST http://127.0.0.1:8000/ingest
```

当前期望规模：

```json
{
  "documents": 511,
  "chunks": 8708
}
```

## 4. 回答结果怎么理解

当前系统返回时通常有两层：

### A. top chunk / evidence

表示系统当前最优先拿到的证据片段。

### B. source / retrieval_source

常见值：

- `entity_link`
  - 通常说明已经稳定命中了主实体来源
- `lexical`
  - 通常说明先靠术语 / alias / curated anchor 命中

### C. source_id

常见值：

- `diablo2-io`
- `arreat-summit`
- `purediablo-d2wiki`
- `curated-anchor`

其中：

- `curated-anchor` 代表系统为社区黑话 / 缩写专门补的稳定锚点层

## 6. 当前能力边界

请把当前系统理解成：

> **“可用的 Diablo II 百科 / 社区术语问答底座”**

而不是：

> **“完整成熟的全自动游戏顾问”**

所以你现在最适合用它来：

- 查概念
- 查黑话
- 查缩写
- 查基础 build archetype
