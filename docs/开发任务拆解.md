# Diablo II 知识库 / GraphRAG 开发任务拆解

这份文档把后续开发拆成**可以逐步执行的 backlog**。  
原则：先做能跑通的主链路，再做效果优化。

---

## Phase 0：冻结基线

### 目标
确保后续开发全部基于当前 merged 主层，不再来回切换数据入口。

### 任务
1. 确认后续主入口固定为 `docs/tier0/merged/`
2. 确认规范固定为 `docs/data-spec-v1.md`
3. 确认开发主说明固定为 `docs/README.md`
4. 确认生产流程固定为 `docs/production-flow.md`

### 完成标准
- 团队内部只讨论 merged 主层
- 不再以 base/high-value/purediablo 单独作为生产入口

---

## Phase 1：做可运行的最小问答原型

### 目标
让系统先具备最基本的问答能力。

### 任务
1. 读取 `merged/canonical-entities.jsonl`
2. 读取 `merged/canonical-claims.jsonl`
3. 读取 `merged/provenance.jsonl`
4. 读取 `merged/chunks.jsonl`
5. 做一个本地检索器：
   - alias lookup
   - canonical entity lookup
   - chunk retrieval
6. 做一个最小问答接口：
   - 输入问题
   - 返回答案 + 来源

### 优先级
最高

### 完成标准
- 可以问 item / runeword / skill / area / monster 基础问题
- 输出包含来源 URL

---

## Phase 2：Alias / Canonical 检索链路

### 目标
解决中英文混合问句的第一跳召回问题。

### 任务
1. alias normalization
2. 中文 alias 扩展
3. 英文 alias 扩展
4. canonical entity 精确匹配
5. entity fallback 搜索

### 重点
- 先别调 embedding
- 先把 alias 命中率做起来

### 完成标准
- “Spirit / 精神 / 军帽 / 地穴 / 火炬”都能稳定命中正确实体

---

## Phase 3：Chunk 检索层

### 目标
让问答有正文证据，不只是图节点。

### 任务
1. 基于 `merged/chunks.jsonl` 建 lexical 检索
2. 给 chunks 建 embedding
3. 做 hybrid retrieval：
   - alias
   - lexical
   - embedding
4. 用 provenance 把 claim 和 chunk 联起来

### 完成标准
- 每个答案都能给出对应 chunk 证据

---

## Phase 4：GraphRAG 图扩展链路

### 目标
让系统不是只查文本，还能做结构化扩展。

### 任务
1. 导入 merged graph 到 Neo4j
2. 接入：
   - nodes
   - edges
   - claims
   - canonical_entities
   - canonical_claims
   - provenance
3. 做 entity neighborhood expansion
4. 做 source-aware rerank

### 完成标准
- 能围绕一个 entity 扩展相关知识
- 能输出 graph + text 混合答案

---

## Phase 5：版本与冲突治理

### 目标
避免 D2 / LoD / D2R 混答。

### 任务
1. 把 `version-tags` 接到检索流程
2. 把 `contradiction-seeds` 接到检索流程
3. 回答时区分：
   - classic
   - lod
   - d2r
4. 给版本差异问题单独做过滤器

### 完成标准
- 问 D2R 时不会优先召回 LoD 老资料
- 问版本差异时能显示冲突或差异说明

---

## Phase 6：中文高质量数据继续扩充

### 目标
提升中文问答质量，而不是只靠英文底座。

### 优先级
1. `91d2.cn`
2. `ttbn.cn`
3. `diablo2.com.cn / bbs.diablo2.com.cn`

### 任务
1. 中文 FAQ 抓取
2. 中文 build / 开荒 / MF 指南抓取
3. 中文术语映射
4. 中文地图 / 掉落 / 任务资料
5. 中文内容并入：
   - aliases
   - provenance
   - chunks

### 完成标准
- 中文问句不再高度依赖英文页面才能答对

---

## Phase 7：Embedding 优化

### 目标
提高复杂问句和模糊问句效果。

### 任务
1. 分开两类 embedding：
   - entity embedding
   - evidence embedding
2. 测试中英文混合 embedding 效果
3. 调整 chunk 长度和 overlap
4. 做 rerank

### 完成标准
- 同义词、模糊问法、长问句效果提升

---

## Phase 8：评测体系

### 目标
让优化有客观标准。

### 任务
1. 建评测集
   - 中文 50~100 条
   - 英文 50~100 条
2. 分类：
   - item
   - skill
   - runeword
   - monster / area
   - quest
   - build
   - version diff
3. 评估：
   - alias 命中率
   - entity 命中率
   - claim 命中率
   - evidence grounding
   - answer correctness

### 完成标准
- 每次改动可量化对比

---

## Phase 9：生产化

### 目标
让系统稳定可重复运行。

### 任务
1. 增量抓取
2. 增量清洗
3. 增量 embeddings
4. 增量 graph rebuild
5. 周期性验证
6. 日志 / 报警 / checksum

### 完成标准
- 新数据加入不会破坏旧系统

---

## 推荐执行顺序（最重要）

如果你现在一个阶段一个阶段做，最推荐顺序是：

1. **Phase 1：最小问答原型**
2. **Phase 2：Alias / Canonical**
3. **Phase 3：Chunk hybrid retrieval**
4. **Phase 4：GraphRAG 图扩展**
5. **Phase 5：版本与冲突治理**
6. **Phase 6：中文补强**
7. **Phase 8：评测**
8. **Phase 7：Embedding 优化**
9. **Phase 9：生产化**
